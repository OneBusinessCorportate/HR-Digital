# OneBusiness HR Digital — Recruitment MVP

Внутренняя платформа управления наймом для OneBusiness. Заменяет ручную
Google-таблицу Инги и делает весь процесс найма прозрачным и измеримым — от
первого контакта до конца испытательного срока.

Интерфейс на русском языке. Часовой пояс бизнес-логики — `Asia/Yerevan`.

## Стек

- **Next.js 15** (App Router, React 19, TypeScript) — серверные компоненты и Server Actions
- **Supabase** — PostgreSQL, Auth, Row Level Security
- **Tailwind CSS** — зелёная палитра OneBusiness
- **Zod** — серверная валидация всех форм
- **Vitest** — модульные тесты бизнес-логики

## Быстрый старт

```bash
npm install
cp .env.example .env.local   # заполните значения (см. ниже)
npm run dev                  # http://localhost:3000
```

### Переменные окружения

| Переменная | Назначение |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | URL проекта Supabase |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Публичный (publishable) ключ — можно показывать в браузере |
| `SUPABASE_SERVICE_ROLE_KEY` | Только сервер. Не обязателен для MVP (импорт работает через сессию пользователя) |
| `TEST_PASS_THRESHOLD` | Порог прохождения теста в % (по умолчанию `70`). БД-настройка имеет приоритет |
| `NEXT_PUBLIC_APP_TIMEZONE` | Часовой пояс бизнес-логики (`Asia/Yerevan`) |

Секреты (service-role) остаются только на сервере и никогда не попадают в браузер.

## База данных

Миграции в `supabase/migrations/` (аддитивные, ничего существующего не удаляют):

- `0001_recruitment_schema.sql` — enum'ы, таблицы, ограничения, индексы, триггеры
- `0002_rls_policies.sql` — Row Level Security и функции авторизации
- `0003_settings_seed.sql` — настройки + профили для существующих пользователей
- `0004_harden_functions.sql` — фиксация `search_path` у функций

Применение через Supabase CLI: `supabase db push`, либо выполнить файлы по порядку.

- `supabase/seed.sql` — **демо-данные** (8 кандидатов на разных этапах, включая
  отклонённого и завершённые испытательные сроки). Повторяемый и идемпотентный.
  **Не запускать на проекте с реальными данными.**
- `supabase/dev_users.sql` — **демо-пользователи** (по одному на роль).
- `supabase/import_template.csv` — шаблон для импорта из таблицы.

## Роли и безопасность

- Вся работа с данными кандидатов требует аутентификации.
- Авторизация проверяется на сервере (RLS + серверные проверки роли), роль/ID из
  браузера не используются как источник истины.
- `hr` и `admin` могут редактировать; `manager` — только чтение (аналитика).
- RLS включён на всех таблицах; редактирование разрешено функцией `hr_can_edit()`.
- Все формы валидируются на сервере (Zod); пользовательский текст выводится
  безопасно (React экранирует по умолчанию).

## Воронка найма

`Первый контакт → Тест → Отбор → Собеседование → Оценка опыта → Оффер →
Испытательный срок → Принят`, плюс терминальный статус `Отклонён` (возможен на
любом этапе). Статусы хранятся как enum, не как произвольный текст. Каждая смена
этапа создаёт запись в истории; история никогда не перезаписывается.

## Страницы

- **/dashboard** — метрики за период (фильтры: период/источник/ответственный/должность),
  конверсии, эффективность источников, «требуют внимания», объём работы HR
- **/funnel** — Kanban-воронка с надёжной сменой этапа
- **/candidates** — список с поиском, фильтрами, пагинацией
- **/candidates/[id]** — карточка кандидата: профиль, таймлайн, все действия
- **/candidates/new** — быстрая форма добавления с проверкой дубликатов
- **/interviews** — предстоящие/проведённые/без результата, предупреждения
- **/reports** — объём работы по сотрудникам и эффективность за период
- **/import** — импорт из CSV с предпросмотром, валидацией и защитой от дубликатов

## Метрики

Вся бизнес-логика — в `src/lib/domain/` (чистые функции, покрыты тестами). Метрики
считаются из реальных данных БД:

- **Объём работы** (`computeWorkVolume`) — по зафиксированным действиям (журнал
  активности), а не по текущему этапу.
- **Конверсии** (`computeConversions`) — по каждому переходу воронки, с явными
  числителем и знаменателем, деление на ноль исключено, формат `25% — 5 из 20`.
- **Удержание первого месяца** (`firstMonthRetention`) — единое определение,
  используется одинаково на всех страницах.
- **Эффективность источников**, **среднее время на этапах**, **время до
  собеседования/найма** — там же.
